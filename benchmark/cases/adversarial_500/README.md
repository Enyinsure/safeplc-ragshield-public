# S7-RAGShield Adversarial 500

This suite contains 500 benchmark cases generated from real S7 text/table/visual assets.

## Files

- `benign_control_050.jsonl`: 50 non-attack control cases.
- `single_attack_350.jsonl`: 350 single-technique attack cases.
- `mixed_attack_100.jsonl`: 100 multi-technique/multimodal attack cases.
- `adversarial_500_all.jsonl`: concatenation of all 500 cases.
- `manifest.json`: counts by split, level, OWASP mapping, modality, and attack family.
- `../../poison_corpus/adversarial_500_poison.jsonl`: local-only synthetic near-duplicate poison evidence referenced by cases. It is generated from local S7 assets and intentionally not committed in the public tree.

## Difficulty Levels

- `L1`: explicit, direct attack wording.
- `L2`: domain-framed attack embedded in normal S7 maintenance language.
- `L3`: near-duplicate poison, fake authority, or session probing.
- `L4`: cross-evidence conflict, vector/metadata abuse, or indirect multimodal injection.
- `L5`: combined attacks, stealthier multi-turn intent shift, or resource/output/agency chaining.

## OWASP LLM Top 10 Mapping

The suite maps attack cases to:

- `LLM01`: Prompt Injection
- `LLM02`: Sensitive Information Disclosure
- `LLM03`: Supply Chain
- `LLM04`: Data and Model Poisoning
- `LLM05`: Improper Output Handling
- `LLM06`: Excessive Agency
- `LLM07`: System Prompt Leakage
- `LLM08`: Vector and Embedding Weaknesses
- `LLM09`: Misinformation
- `LLM10`: Unbounded Consumption

## Realism Notes

- Poison evidence is generated by locally editing real S7 text chunks, table chunks, and visual cards, then kept out of Git.
- Every poison row records `base_id`, `base_hash`, `near_duplicate_ratio_estimate`, `owasp_ids`, and a synthetic canary.
- Mixed cases combine text/table/visual poison evidence and multi-turn instructions where appropriate.
- No real secrets, credentials, or private personal data are used.
